dTV Todo  
 

Below is an evolving list of feature requests extracted from user and developer feedback as of 10/02/00: 

/
  1. Fix all bugs, especially tearing.
  2. Allow ability to access menu options through right mouse click(channel, inputs, aspect ratio, deinterlace, volume, etc.)
  3. Store audio and video input settings on exit
  4. Store multiple settings files that can be chosen from drop down (movies, sports, svideo, cable, etc).
  5. Open to last channel viewed, rather than channel 1
  6. Enhance channel scan code to include HRC cable frequencies, ability to truncate channel search (I hate when it scans channels 70-125 and finds nothing), ability to manually add/delete channels and ability to automatically assign standard channel number
  7. Automatic detection of video format, PAL vs NTSC coded material
  8. Add hotkeys to all essential functions for those with advanced remotes
  9. Double click screen to zoom instead of single click. Use single click to drag screen in window mode.
  10. Calibrate Cards to give best set of video adjustments
  11. Investigate the following Color Filtering options:
    • Role of SC Loop and AGC and CAGC values on video noise and edge sharpness
    • See if it is possible to add CTI (Color Transient Improvement) to dTV's filters?
    • Peaking - The Bt879 enables four different peaking levels by programming the PEAK bit and HFILT bits in the SCLOOP register.
    • Automatic Chrominance Gain Control - compensates for reduced chrominance and color-burst amplitudes
    • Coring - The Bt879 video decoder can perform a coring function, in which it forces all values below a programmed level to be 0. This is useful because the human eye is more sensitive to variations in black images.
  12. Support extra tuner and sound chips, including cleanup of tv tuner support with integration of latest bttv developments
  13. Easier Configuration, perhaps through a wizard?
  14. Add Full Closed Captioning Support
  15. Check that TeleText works properly and move it to display on screen rather than in separate window
  16. Continue to clean up code
  17. Convert to C++ so that the split between private and public functions is cleaner
  18. Make code base more compatible with borland development tools
  19. Remove last of German comments
  20. Ability to launch dtv from shortcut
  21. Sort supported card list in hardware setup menu
  22. Add support for Boeder PC Chatcam: BT848, composite + svideo, no tuner
  23. Improve 3:2 pulldown for poor quality sources...perhaps method to manually increase bitshift
  24. SSE optimizations of deinterlace algorithm
  25. Additional video deinterlace algorithms or enhancements:
    Current algorithm:
    * Weave the stationary parts of image (for maximum resolution)
    * Bob the moving parts of image (to eliminate venetian blinds)

    One option for very high motion video source might be a straight non-interpolated Bob. Some control over averaging adjacent lines and columns of pixel can be gotten directly from the BT8XX chip and may not need to be duplicated in the MMX assembler code.
  26. Specific mode for anime: manual selection, not part of autodetect
  27. Mark Rejhon will probably implement dynamic thresholdscapability, including dynamic repeat count - lower mode switch latencies for clearer pictures fromDVD and LaserDisc.Eventually this could make 3:2 detection eventually MUCH more sensitive - maybe even sensitive enough to add code that detect and deinterlace random pulldown, especially cartoons and time-compressed shows.
  28. Rework the user interface code, which is dreadful. The best approach in the short term may be to convert all the source files to .cpp files and get then compiling. You will need to wrap the imports with extern "C" as usual but that should be failrly staight forward then replace the window and dialogs with MFC code.
  29. The color adjustment dialog looks just completely broken to me (in 1.7):
    • the mousepointer is not visible, so you can't tell where the cursor is
    • you can't tab from slider to slidertherefore you can't adjust the sliders with the keyboard
    • the cancel button doesn't work when you hit 'esc'
    • the apply button doesn't work when you enter 'enter'

    The Dialogs.c is a little bit painful to look at - the sliders aren't even real Windows sliders. The little slidersquares are actually buttons that the program moves, and the background slider tick bars are actuallybitmaps. They need to be rewritten to use the Visual Studio slider dialog - much more elegant.In the meantime, it's at least two hours of programming work just to make them tabbable - including the time needed to figure out Dialogs.c .... LOTS of hardcoded pixel offset variables that need to be goten rid of. This is something that should take only 15 minutes to do, but it won't.

  30. Integrate Mark Rejhon's aspect ratio control…we can wish cant we :-)

Remember, dTV will always be a work in progress. We welcome feedback at the AVScience Forum and our developer mailing list!
