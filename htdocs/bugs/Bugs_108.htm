Known Bugs & Issues 
 

dTV is still in the early stages of development.  It will probably never be truly stable and shrink-warp quality.  That being said the developers are keen to have the software running as well as possible.  This file is intended to be used as a place to keep track of the current known bugs so that people who want to join the project have a good place to start looking for problems to fix.

  1. VBI code sometimes cases video to jump about.  (cause could be constant DMA overruns)
  2. After either pauses, format changes or maximizing the software will no longer lock onto correct pulldown mode and run with bad artifacts.  BT Reset does no cure this but software behaves well after it is stopped then started (again, it appears to be DMA related, or possible bug in the output code causing the flip frames to get out of sync)
  3. Does not work on Mpact video cards in 24 bit hacked mode (not sure if this is possible to fix although DVD software run on these cards)
  4. Playback is jumpy.  This is because there is no code to try and synchronize the TV data to the PV Video card refresh rate.  On slower machines this should be less of a problem but at higher speeds the TV frames will not be displayed smoothly.
  5. Tearing appears.  May be linked to some of the above bugs, in particular number 2.
  6. The tuner operation of the software is extremely buggy. Improvements in this area are most welcome.
  7. Pulldown detection gets confused and doesn't like to go back to video once it's in 3:2 mode.
  8. You may experience overlay errors on a fairly regular basis, especially if you change resolutions or refresh rates, or if you open another program like powerDVD.
  9. Sometimes when you quit the program in s-video mode and restart, dTV will default to tuner but still shows s-video channel.
  10. Some people have faint color bars in the displayed output with some sources. This is probably due to Macrovision and seems to be card-specific.

Feel free to add your own!
