dTV.ini Settings

Many features of the program are configurable through the menu system.  Some advanced settings and functions in beta test will require changes to this ini file to work properly.  All the settings used by the program are stored here and no changes are made to the registry.

[Pulldown]

PulldownThresholdLow Determines when PAL pulldown detection will detect a noticable improvement in combing.  Lower values (more negative) will make detection require more combing to change.  Change this if PAL detection is not working for you.

PulldownThresholdHigh Determines when PAL pulldown detection will go back from movie mode to video mode.  This value copes with certain types of video noise, if it is set too low then it will flip back on static movie images, if set too high it will remain in movie mode for too long.

PulldownRepeatCount This is the number of times the pulldown pattern must be detected before switching.

Threshold32Pulldown  This is the number of pixels per field that need to be similar for NTSC movie detection to detect a triplet of fields from the smae movie frame.  Higher values will cope with more video noise but will not detect the movies as well.

BitShift This is used by the NTSC detection code to limit what pixels are the same.  This should allow for noise in source.  The value is applied to the square of the pixel difference with the results bitshifted right by this amount.

DiffThreshold .Not Used

PulldownRepeatCount2 The number of times an unexpected pattern can occur before switching back to video video mode from a film mode.

[Deinterlace]

EdgeDetect This values governs the strength of edge detection, use higher values if you think that the video deinterlacing is occurring too often.

JaggieThreshold This values governs the threshold for fine line dtection, use higher values if you get problems with small text
